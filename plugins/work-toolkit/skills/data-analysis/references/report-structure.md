# Report Structure

Use this default Markdown structure unless the user asks for a different format.

```md
# <Title>

## Executive Summary
- 2-5 concise bullets with the most decision-relevant findings

## Objective and Scope
- question to answer
- time range
- important assumptions

## Data or Material Overview
- what inputs were used
- any major caveats

## Key Findings
### 1. <Finding title>
Evidence, chart or table, and interpretation

### 2. <Finding title>
Evidence, chart or table, and interpretation

## Detailed Analysis
Use subsections as needed for trends, comparisons, segments, or source themes.

## Conclusions and Recommendations
- conclusion
- recommendation
- next validation step if uncertainty remains

## Limitations and Risks
- data or source gaps
- scope limits
- interpretation limits

## Appendix
### Sources
### SQL
### Method Notes
```

## Writing rules

- Make section titles informative.
- Make chart titles conclusion-oriented, not field-name-oriented.
- Put the evidence close to the claim it supports.
- Keep executive summaries short and concrete.
- Distinguish observed facts from interpretation.
